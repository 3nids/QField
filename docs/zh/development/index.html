开发与支持¶

许可¶

QField是根据GNU公共许可证（GPL）Version2或更高版本发布的。在此许可证下开发QField意味着您可以检查和修改源代码，并保证您始终可以访问基于QGIS的现场数据收集应用程序，该应用程序免费且可以自由修改。

您可以在此找到源码 on github

支持¶

你可以再此提问 gis.stackexchange

你可以使用 user mailing list

我们还提供商业支持, 联系我们

让它成长，让它成为你自己所有¶

这是一个功能列表，我们认为这些功能对于良好的用户体验和有效的工作流非常有用。

编辑部件¶

编辑小部件是高效现场工作的核心部分。在触摸设备上输入属性是花费时间最多的工作之一，保持这项任务尽可能简单是使工作愉快的关键。

一些重要信息：ref:编辑字段小部件已可用 but more is possible`例如在QGIS<http://gis.stackexchange.com/a/30051/9839>中。

说明¶

对于一个漂亮的地图，a legend是一个非常有用的工具。它给用户的上下文的语义意义不明白的笔画形状，颜色，和其他视觉编码。

它还可以用于访问单个层的附加元数据或选择要处理的层。

基于/脱机同步的连接¶

数据收集是QField提供重要用途之一。但数据往往不会只存储于设备，而通常会储存于数据库。目前，直接在数据库上工作是可以的，但由于在现场时网络访问通常不可靠，您可以在台式计算机上进行手动同步。

如果在设备上运行同步进程，那么当网络可用时，数据将自动与服务器同步，而当没有可用的网络连接时，将在本地缓存数据，这将非常好。

可以与 GeoGIG 联动或其他服务器组件，如数据库。

提高坐标精度¶

计算机主要使用两种不同的数据类型在内部存储小数（浮点数）。` 单精度<https://en.wikipedia.org/wiki/single-precision_floating-point_format>`_ and` 双精度<https://en.wikipedia.org/wiki/double-precision_floating-point_format>`` precision.对于单精度，只有数字的前6位是有效的，可以信任；对于双精度，前15位是可信任的。这将考虑所有数字（小数点之前和之后）。

当使用坐标时，当需要高精度时，这一点非常重要。只有6个有效数字通常意味着一米左右的最大精确度，但取决于CRS和地球表面的位置。

目前，QField的某些部分使用单点精度处理坐标。通过修正这一缺点，可以充分利用精确定位装置的全部精度，这将是一个很好的改进。

GPS进入省电状态信息的属性¶

定位设备提供的信息不仅仅是纬度和经度。他们还能够估计目前能够提供的精度。该信息可以是根据卫星数量等变量计算的半径，也可以是RTK状态等参数的附加标志。

编辑几何图形¶

QField提供数字化新几何图形、编辑现有属性或删除特征。它缺少的是编辑现有几何图形的功能。

这可以分割成不同的部分，如移动点、移动其他特征、编辑顶点、移动边…

搜索功能¶

在搜索字段的帮助下定位功能确实有助于提高工作效率。一般来说，这可以分为两类：

基于属性的搜索

搜索结果是基于项目中图层上的要素属性提供的。可以使用本地数据库进行搜索。内部数据基于您使用的自定义数据，或者直接查询层以查找具有相关属性的功能。

地理编码

根据实际地址提供搜索结果。像这样的服务`Nominatim <http://nominatim.openstreetmap.org/>`_ 可以通过Web界面访问，并将实际地址转换为地图上的位置。尝试链接。

外部测量装置¶

您是否使用带有传感器的外部设备？气压，温度，坡度，一氧化碳浓度，或者其他的…不必从外部设备读取信息并手动再次输入，这不是很好吗？我们希望通过将外部测量设备直接连接到QField的支持，使您的生活更轻松。

录音¶

有没有想过，当你在野外使用触摸屏时，如何录入大量的文字？

不要这样！

有可能像图像集成一样进行音频集成，这将是令人惊讶的。回到办公室后，你可以用普通键盘输入信息。

点击打开QGIS文件¶

只需在QField中通过在文件管理器中单击打开一个QGIS项目。
